// 01 — Overview

What We Built

We designed and built a working autonomous robot prototype intended for last-mile food delivery on structured campus environments. The system combines real hardware sensor integration on a Raspberry Pi 4 / FreeNove Mecanum-wheel platform with a full AI architecture proposal covering perception, mapping, and intelligent path planning.

The robot was programmed with three live autonomous modes — ultrasonic obstacle avoidance, infrared line tracking, and light tracing — and demonstrated live with the FreeNove WiFi GUI. The broader AI pipeline (CNN + RNN + Reinforcement Learning) was designed and presented as the production deployment architecture for a real campus delivery system.

// 02 — Sensor Systems

Four Autonomous Behaviors

🔊

Ultrasonic Obstacle Avoidance

HC-SR04 sensor on a servo motor sweeps 30°, 90°, and 150°. At each angle, distance is measured and compared to a 30 cm threshold. The robot automatically moves forward, turns, or reverses and U-turns based on which directions are blocked.

🔦

Infrared Line Tracking

A 3-channel IR sensor array (Left / Middle / Right) is mounted below the chassis. Each sensor returns HIGH on black, LOW on white. Five-state directional logic (full left → straight → full right) keeps the robot precisely on a marked path.

💡

Light Tracing

Dual photoresistor sensors compare left and right light voltages in real time. The robot steers toward the brighter light source, enabling it to home in on illuminated delivery targets or follow directional light signals.

🎮

Omnidirectional Motor Control

Four independent Mecanum wheels allow forward, backward, lateral strafe, and in-place rotation — no multi-point turns needed. Each motor is controlled via setMotorModel(A,B,C,D) with values from −4096 to +4096 for full speed control.

# Motor control — Mecanum wheel movement def setMotorModel(A, B, C, D): # A = Left Front | B = Right Front # C = Left Rear | D = Right Rear # +4096 = full forward | -4096 = full reverse | 0 = stop PWM.setMotorModel(A, B, C, D) # Ultrasonic obstacle avoidance decision logic def avoid_obstacle(dist_left, dist_mid, dist_right): if dist_left > 30 and dist_mid > 30 and dist_right > 30: setMotorModel(2000, 2000, 2000, 2000) # Forward elif dist_left < 30 and dist_mid < 30 and dist_right > 30: setMotorModel(2000, -2000, 2000, -2000) # Turn Right elif dist_left > 30 and dist_mid < 30 and dist_right < 30: setMotorModel(-2000, 2000, -2000, 2000) # Turn Left else: setMotorModel(-1500, -1500, -1500, -1500) # Reverse

Decisions

  • Path planning
  • Behavior pred.

The CNN handles visual understanding of the environment — identifying pedestrians, segmenting navigable surfaces, and tracking moving objects. The RNN fuses all sensor data to build and maintain a real-time campus map. The Reinforcement Learning agent uses this map to select safe, efficient delivery routes and predict pedestrian behavior dynamically.

// 05 — Business Model

Competitive Advantages

Advantage Description
Compact & Maneuverable Mecanum wheels provide full omnidirectional movement — navigates narrow sidewalks, courtyards, and corridors without multi-point turns.
Fully Autonomous CNN + RNN + Reinforcement Learning handles all navigation decisions. No human operator required at any point in the delivery loop.
Low Operating Cost Electric, lightweight, no labor scaling. Cost per delivery decreases as fleet grows — unlike human courier models.
Safe & Contactless Smart obstacle avoidance + sealed compartment = hygienic, accident-resistant delivery. Ideal for medical campuses and crowded student areas.
Scalable by Design API-ready for integration with existing campus food ordering apps. Adding more robots requires zero additional staffing or infrastructure.
// 06 — Legal & Ethics

Responsible Deployment

⚖️ Liability

Clear legal frameworks must define responsibility for collision and property damage scenarios before deployment on any campus. Insurance and liability frameworks must be established with campus administrators.

🍽️ Food Safety

Robots must comply with temperature control standards, contamination prevention regulations, and local health department certification requirements for food transport.

♿ ADA Compliance

Robots must never obstruct accessible ramps, entrances, or emergency exits. Navigation systems must be explicitly programmed to respect ADA-required clearance zones at all times.

🔒 Privacy by Design

All sensor and camera data is used solely for navigation. No facial recognition, no behavioral profiling, no data retained beyond the scope of a single delivery mission.
